📖 Comment fonctionne la détection d'anomalies de trésorerie ?

🧠 L'algorithme

Le module utilise IID Spike Detection de ML.NET sur les opérations comptables. Il calcule un score statistique pour chaque mois et identifie les valeurs qui s'écartent significativement du comportement normal des flux bancaires.

📊 Les métriques

  • Flux Net — Détecte les mois où le solde net (entrées − sorties) est anormalement haut ou bas
  • Encaissements — Repère les pics inhabituels dans les réceptions bancaires (AE, BEE)
  • Décaissements — Identifie les sorties de fonds anormales (BSE, CS)
  • Solde Disponible — Détecte les variations anormales de la balance disponible (AE, BS, BSR, BEE, CS)
  • Solde Réel — Signale les écarts anormaux du solde réel confirmé (AE, BSE, BEE, CS)
  • Solde Prévisionnel — Identifie les anomalies sur la projection globale (Réel − Émis + Encaissé)
  • Volume — Détecte les mois avec un nombre d'opérations inhabituel
  • Frais Bancaires — Signale les anomalies sur les frais bancaires purs (codes C01–C07 uniquement) :
    • C01 — Tenue de compte
    • C02 — Commission
    • C03 — Frais extrait
    • C04 — Intérêt débiteur
    • C05 — Message
    • C06 — TVA
    • C07 — Autres frais bancaires

⚙️ Les paramètres

  • Monnaie — CDF, USD ou EUR pour une analyse ciblée par devise. Sans filtre, les montants sont convertis en CDF via les taux de change du jour.
  • Confiance (%) — 90 % = plus sensible, 95 % = équilibré, 99 % = très strict

📈 Lire les résultats

  • Ligne bleue — Valeurs normales
  • Points rouges — Anomalies détectées
  • Score — Amplitude de l'écart (plus élevé = plus marqué)
  • P-value — Probabilité que le point soit normal (plus basse = plus certain)

� Colonnes du tableau d'anomalies

  • Période — Mois concerné par l'anomalie (format mois/année)
  • Valeur — Montant réel observé pour ce mois sur la métrique sélectionnée
  • Score — Amplitude de l'écart par rapport au comportement normal. Plus le score est élevé, plus le pic est prononcé
  • P-value — Probabilité statistique que ce point soit normal. Plus la p-value est basse, plus l'anomalie est certaine (ex : 0.01 = 1 % de chance d'être normal)

💱 Conversion de devise

  • Avec un filtre de devise (CDF, USD, EUR) — les montants sont affichés dans la devise sélectionnée, sans conversion.
  • Sans filtre (« Toutes ») — les montants USD et EUR sont convertis en CDF via les taux de change de la table Taux (base SUIVI_TP_DF), en utilisant l'USD comme devise intermédiaire.
  • Le taux utilisé est celui du 1er du mois, ou à défaut le plus récent avant cette date (même logique que le système de taux du jour).

💡 Conseils

  • Les anomalies de flux net peuvent révéler : régularisations massives, erreurs comptables, ou opérations exceptionnelles.
  • Les frais bancaires ne couvrent que les codes C01–C07 (pas les débits d'opérations C10, E01).
  • Un minimum de 4 mois est requis. Plus la fenêtre est large, meilleure est la détection.
